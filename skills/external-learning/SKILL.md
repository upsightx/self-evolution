# External Learning(v2 - 两阶段深度学习)

从外部世界主动学习,产出可操作的知识而非浅层摘要。

## 核心原则

- **宁少勿滥**:10 条深度笔记 > 100 条标题摘要
- **两阶段**:先广筛、后深读,分开执行
- **原文存档**:高价值条目保存原文,可回溯
- **结构化入库**:关键发现写入 memory_db,支持后续检索

## 信息源

|--------|--------|--------|------|----------|

## 第一阶段:广筛(子Agent并行)

### 目标
快速扫描信息源,产出候选清单。**不写摘要,只筛选。**

### 子Agent配置
- **model**: MiniMax(默认)
- **mode**: run
- **timeout**: 300 秒
- **每次至少 3 个子Agent**,优先级 1 必选,其余按频率轮换

### 子Agent指令
读取 `references/base-template-v2.md`(通用规则)+ 对应 `references/{源}.md`(数据源配置),拼接为完整指令。

### 子Agent输出格式
写入 `memory/learning/candidates-{源}-{date}.md`:

```markdown
# {源名称} 候选清单 {date}

| # | 标题 | URL | 分数 | 一句话理由 |
|---|------|-----|------|-----------|
| 1 | xxx | url | 8.5 | 为什么值得深读 |
| 2 | xxx | url | 7.0 | 为什么值得深读 |
```

**只输出分数 ≥ 5 的条目,最多 15 条,按分数降序。**

### 执行流程
1. 检查 `memory/heartbeat-state.json` 的 `lastChecks`,判断哪些源需要更新
2. 并行派发子Agent
3. `sessions_yield` 等待完成
4. 收集所有 candidates 文件

## 第二阶段:深读(主Agent执行)

### 目标
对候选清单中 **分数 ≥ 7** 的条目,逐条深读原文,产出真正有价值的知识笔记。

### 来源等级标签(新增,强制)
每条深读笔记必须明确标注来源等级,禁止混写:

- **摘要级**:只读了 arXiv abs / PapersWithCode / HuggingFace Papers / 新闻摘要页 / README 首页等二手或摘要材料
- **原文级**:读了论文 PDF 正文、完整技术博文、完整文档正文、完整 README / docs 主体
- **多源验证级**:在原文级基础上,额外做了至少一次交叉验证(如论文 + 代码仓库 / 官方文档 + 新闻稿 / 两个独立来源)

**硬规则**:
- 未打开 PDF/正文,不得标为"原文级"
- 未做交叉验证,不得标为"多源验证级"
- 如果只能拿到摘要页,必须诚实标"摘要级"
- 对论文类条目,**分数 ≥ 8 默认要求至少做到原文级**;做不到时,必须在笔记中说明原因

### research 任务二次验证(新增,强制)
对以下条目,深读完成后必须追加一次二次验证,未验证不算完成:
- 论文 / 技术报告
- 涉及 benchmark、SOTA、准确率、融资金额、用户规模等关键数字的条目
- 任何会被写成"对我们的启发"的高价值研究条目

二次验证至少满足以下之一:
1. **交叉来源验证**:再读一个独立来源核对核心结论或关键数字
2. **反证检查**:主动检查论文/项目的局限性、失败案例、作者自述 limitations
3. **实现核对**:去代码仓库 / 官方文档确认论文里说的方法、工具或数据是否真的存在

笔记中必须显式写出:
- **来源等级**:摘要级 / 原文级 / 多源验证级
- **二次验证**:做了什么验证;如果没做,为什么没做

### 执行流程

1. **合并去重**:读取所有 `candidates-*-{date}.md`,合并去重,按分数排序
   - 去重规则:标题相似度 > 80% 或 URL 域名+路径相同,视为同一条目
   - 保留分数最高的那条,标注"多源命中: github+pwc"等
   - **已读过的条目**:检查 `memory/learning/` 历史笔记,若标题已出现过则跳过深读(标注"已读")
2. **筛选深读列表**:取分数 ≥ 7 的条目(通常 10-20 条)
3. **逐条深读**:
   - 用 `web_fetch` 读取原文(论文摘要页、GitHub README、新闻全文)
   - 提炼以下结构化信息:

```markdown
### [标题](url)
**来源**: {源} | **分数**: X.X | **日期**: YYYY-MM-DD
**来源等级**: 摘要级 / 原文级 / 多源验证级
**二次验证**: [做了什么验证;若无则写"未做,并说明原因"]

**核心内容**(3-5句,不是复述,是提炼关键信息):
- 这个东西解决什么问题
- 用什么方法/技术
- 效果如何(具体数据)

**对我们的启发**(1-2句):
- 能怎么用 / 对现有工作的影响

**关键数据**:
- 具体指标、benchmark 数字、star 数等

**落地评估**:
- 相关模块:{填写具体模块名,如 evolution_strategy / feedback_loop / memory_retrieval,无则填"无"}
- 改动规模:小改(<20行) / 中改(20-200行) / 大改(>200行) / 不确定
- 前置条件:无 / 需要新依赖 / 需要架构变更
- 落地优先级:P0(立即可做) / P1(本周可做) / P2(需要讨论) / skip(不适用)

**原文存档**: memory/learning/raw/{date}/{源}-{序号}.md
```

4. **保存原文**:web_fetch 的原文内容保存到 `memory/learning/raw/{date}/` 目录
5. **写入正式笔记**:`memory/learning/{date}.md`
6. **结构化入库**:分数 ≥ 8 的条目写入 memory_db(add_observation type=discovery)
7. **清理候选文件**：删除 `candidates-*-{date}.md`

### 落地评估规则（深读完成后执行）

**过滤噪声（硬规则）：**
- 相关模块 = "无" → 落地优先级强制设为 skip
- 前置条件 = "需要架构变更" → 落地优先级最高只能 P2
- 无开源代码且无可验证数据 → 落地优先级降一级
- 纯理论/纯学术/跟 Agent 无直接关系 → skip

**触发机制：**
- **P0 条目**：自动调用 `evolution_executor.create_experiment()` 创建 draft 实验，下次心跳提醒主人确认
- **P1 条目**：写入 `memory/learning/landing-candidates.md` 待办清单，周审计时汇总
- **P2 条目**：只记录在深读笔记中，不主动推送
- **skip 条目**：不做任何额外处理

**落地评估汇总：**
每次深读完成后，在正式笔记末尾追加一个汇总：

```markdown
## 落地评估汇总
| 条目 | 相关模块 | 优先级 | 状态 |
|------|----------|--------|------|
| MiroThinker | evolution_strategy | P0 | → draft实验 #N |
| xxx | feedback_loop | P1 | → 待办清单 |
| yyy | 无 | skip | — |
```

### 深读数量控制
- 单次深读上限 **15 条**(避免 token 消耗过大)
- 如果候选超过 15 条,只取 TOP 15
- 融资类条目不需要深读原文(API 结构化数据已足够),直接整理即可

## 第三阶段:输出

### 正式笔记格式
`memory/learning/{date}.md`:

```markdown
# 外部学习笔记 YYYY-MM-DD

## 深度笔记(N条)
[逐条深读结果]

## 融资速览(N条)
[融资条目表格]

## 候选但未深读(N条)
[分数 5-6.9 的条目,仅标题+URL+一句话,供后续按需查看]

## 趋势观察
[基于本次学习的 2-3 条趋势判断]
```

### 飞书同步
用 `feishu_create_doc` 创建文档:
- **folder_token**: `<your_folder_token>`
- **标题**: `外部学习笔记 YYYY-MM-DD`
- 深度笔记用 callout 高亮,融资用表格,趋势用 callout

### 通知决策
**主动通知**(满足任一且在 08:00-23:00):
- 分数 ≥ 9 的条目
- 与主人业务直接相关的重大信息

**静默记录**:常规更新、深夜时段(23:00-08:00)

## 质量标准

### 深度笔记必须满足
- [ ] 读过原文,不是只看标题猜的
- [ ] "核心内容"包含具体方法/数据,不是空话
- [ ] "对我们的启发"是具体可操作的,不是"值得关注"
- [ ] 原文已存档到 raw/ 目录

### 禁止
- ❌ 没读原文就写摘要
- ❌ "值得关注"、"意义重大"等空话
- ❌ 复述标题当摘要
- ❌ 丢弃原文只留摘要
