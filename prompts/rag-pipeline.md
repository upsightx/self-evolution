# RAG Pipeline Prompt

检索增强生成的完整 prompt 链。

## 1. 意图分类

```
判断用户查询的意图类型，返回以下之一：
- direct: 直接回答（闲聊、帮助）
- rag: 需要检索数据后回答
- stats: 需要统计分析
- help: 系统帮助

只返回意图类型，不要解释。
```

规则优先（关键词匹配，零成本）：
```python
INTENT_RULES = {
    "direct": ["你好", "谢谢", "帮助", "你是谁"],
    "stats": ["统计", "多少次", "频率", "排行", "占比"],
    "help": ["怎么用", "功能", "教程"],
}
# 都不匹配 → 默认 rag，或走 LLM 分类
```

## 2. 检索后生成

```
你是一个智能助手，基于检索到的数据回答用户问题。

强制要求：
- 引用具体数据时标注来源
- 数据不足时明确说明，不要编造
- 优先用数据说话，少用推测

用户查询：{query}

检索到的数据：
{context}

请基于以上数据回答。
```

## 3. 无数据兜底

```
抱歉，没有找到与"{query}"相关的数据。

可能的原因：
1. 时间范围内没有相关记录
2. 关键词需要调整

建议：
- 尝试扩大时间范围
- 使用更具体或更宽泛的关键词
```
