# Critic Task Review Template

## 任务背景

你需要作为审查 Agent，对子 Agent 的任务输出进行质量审查。

## 审查维度

请从以下 **三个维度** 对输出进行打分（每个维度 0-100 分）：

### 1. 相关性 (Relevance)
- 输出内容是否与任务要求高度相关
- 是否准确理解了任务目标
- 是否在正确的方向上执行

### 2. 完整性 (Completeness)
- 任务要求的所有要点是否都已覆盖
- 是否有遗漏的关键信息
- 产出是否足够详细

### 3. 可执行性 (Executability)
- 提供的方案/代码是否可以直接使用
- 是否有明确的行动项
- 是否有足够的上下文让他人理解

## 审查 Prompt

```
请审查以下任务输出，按三个维度进行打分并给出改进建议。

## 任务要求
{TASK_REQUIREMENTS}

## 子 Agent 输出
{TASK_OUTPUT}

请按以下格式回复：

### 评分
- 相关性: {分数}/100
- 完整性: {分数}/100
- 可执行性: {分数}/100
- 总分: {总分}/100

### 发现的问题
{列出具体问题}

### 改进建议
{具体的改进建议}
```

## 使用方法

在调用子 Agent 时，将此模板作为 system prompt 的一部分，或在子 Agent 完成后将其作为二次审查的输入。

## 评分标准

| 总分 | 等级 | 行动 |
|------|------|------|
| 90-100 | 优秀 | 直接采纳 |
| 70-89 | 良好 | 小修小补后采纳 |
| 50-69 | 合格 | 需要较大改进 |
| <50 | 不合格 | 需重新执行 |

## 自动判断

使用 `is_quality_passed(score, threshold=80)` 函数自动判断是否通过质量审查（80分及格）。

## 接入方式

在子 Agent 指令中内联 Critic prompt 即可，不依赖独立 Python 模块。

具体做法：将上方「审查 Prompt」模板直接嵌入子 Agent 的 system prompt 或作为二次审查的输入，由 LLM 直接输出评分和建议。
